## Чек-лист быстрой проверки ⚡

### ✅ Шаг 1: Установки (Equipment)
1. Зайти как Админ
2. Перейти на вкладку **Установки**
3. **ПРОВЕРЬТЕ**: 
   - [ ] Список НЕ ПУСТ
   - [ ] Видны: Бауман-100, Бауман-80, Виброрам, Генератор
   - Если пусто → ошибка, выполните `npm run seed`

### ✅ Шаг 2: PDF Preview
1. Перейти на вкладку **Отчеты**
2. Найти любой отчет
3. Нажать **"PDF Предпросмотр"**
4. **ПРОВЕРЬТЕ**:
   - [ ] Диалог не переполняется
   - [ ] Кнопки (Печать, Скачать, Закрыть) остаются видны
   - [ ] Нет ошибок в консоли (F12)

### ✅ Шаг 3: Редактирование отчета
1. На вкладке Отчеты нажать кнопку **редактирования** (карандаш)
2. **ПРОВЕРЬТЕ что видно**:
   - [ ] **Установка** - поле должно быть заполнено
   - [ ] **Забитые сваи** - раздел видим
   - [ ] **Лидерное бурение** - раздел видим
   - [ ] **Причины простоев** - раздел видим

Если хоть одного поля нет → **ОШИБКА** ❌

### ✅ Шаг 4: Фильтр по датам
1. На вкладке **Отчеты** выбрать:
   - От: **08.04.2026**
   - До: **15.04.2026**
2. Нажать **"Применить"**
3. Список обновился → ✅
4. Нажать **"Скачать PDF"** → PDF генерируется без ошибок → ✅

### ✅ Шаг 5: Все роли
Для каждой роли выполнить:

**Диспетчер** (dispatcher@pilingtrack.local)
- [ ] Зайти в систему
- [ ] Видны модули: Дашборд, Объекты, Отчеты
- [ ] НЕ видны: Пользователи, Справочники
- [ ] Нет ошибок

**Оператор** (operator@pilingtrack.local)
- [ ] Зайти в систему
- [ ] Видны модули: Дашборд, Мои отчеты
- [ ] Нет доступа к админ-функциям
- [ ] Нет ошибок

## Если что-то не работает 🔧

### Установки пусты (Equipment list is empty)
```bash
# Решение:
npm run seed
# Потом обновить браузер (F5)
```

### PDF не генерируется
```bash
# Проверить логи:
npm run dev
# Ищите ошибки про PDF в консоле сервера
```

### Ошибки в переключении вкладок
```bash
# Очистить браузер кэш:
Ctrl+Shift+Del (Windows/Linux) или Cmd+Shift+Del (Mac)
# Или просто перезагрузить страницу: F5
```

### Не видно поле "Установка" при редактировании отчета
```bash
# Это значит что код не внедрен правильно
# Проверьте файл:
src/components/piling/admin-reports/report-form-dialog.tsx

# Должно быть (строка ~55):
<Select value={formEquipmentId} onValueChange={setFormEquipmentId}>
```

## Что проверять в консоли браузера (F12)

Откройте **F12 → Console** и проверьте:

- ✅ **Зеленые сообщения** - OK
- ✅ **Желтые Warning** - не критично
- ❌ **Красные Error** - ПРОБЛЕМА
- ❌ **404 errors** - файл не найден

**Не должно быть:**
```
❌ Cannot read properties of undefined
❌ Error: Equipment not found
❌ TypeError: equipment is not defined
❌ Failed to fetch from API
```

## Часто встречающиеся проблемы

| Проблема | Решение |
|----------|---------|
| Порт 3000 занят | Выполнить в PowerShell: `Get-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess \| Stop-Process -Force` |
| БД не инициализирована | `npx prisma migrate deploy && npm run seed` |
| TypeScript ошибки | `npm run typecheck` и исправить ошибки |
| Стили не загружаются | Hard refresh: `Ctrl+Shift+R` |

## Команды для разработчиков

```bash
# Основные
npm run dev              # Запустить сервер
npm run seed             # Добавить тестовые данные

# Тестирование
npm run test:interactive # Интерактивный тест
npm run test:unit        # Unit tests
npm run test:e2e         # E2E tests (Playwright)

# Проверка качества
npm run lint             # ESLint
npm run typecheck        # TypeScript strict
npm run test:unit:coverage  # Coverage report

# Docker
docker compose up -d     # Запустить все сервисы
docker compose logs -f app  # Логи приложения
docker compose down      # Остановить
docker compose down -v   # Удалить все данные
```

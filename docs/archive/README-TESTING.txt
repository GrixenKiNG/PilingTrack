### Шаг 1: Подготовка (выполнить в терминале)
```bash

# 2. Запустить приложение

# 3. Открыть в браузере

```

### Шаг 2: Вход в систему
```
Email:    admin@pilingtrack.local
Password: password123
```

### Шаг 3: Быстрая проверка (5-10 минут)

V **ПРОВЕРЬТЕ УСТАНОВКИ (Equipment)**
- Перейту на вкладку "Установки"
- Должны быть видны 5 установок:
  - ✅ Бауман-100
  - ✅ Бауман-80
  - ✅ Виброрам РВ-80  
  - ✅ Сваебой ненаправленного действия
  - ✅ Дизельный генератор

V **ПРОВЕРЬТЕ PDF PREVIEW**
- Перейдите на вкладку "Отчеты"
- Нажмите "PDF Предпросмотр" на любом отчете
- Проверьте кнопки (Печать, Скачать, Закрыть) остаются в видимой зоне

V **ПРОВЕРЬТЕ РЕДАКТИРОВАНИЕ ОТЧЕТА**
- Нажмите кнопку редактирования отчета (карандаш)
- Должны быть видны следующие поля:
  - ✅ **Установка** (Equipment)
  - ✅ **Забитые сваи** (Piles)
  - ✅ **Лидерное бурение** (Drilling)
  - ✅ **Причины простоев** (Downtimes)

V **ПРОВЕРЬТЕ ФИЛЬТР ПО ДАТАМ**
- На вкладке "Отчеты" выберите период 08.04.2026 - 15.04.2026
- Нажмите "Применить"
- Нажмите "Скачать PDF"
- PDF должен генерироваться без ошибок

## ❓ ЧТО ДЕЛАТЬ ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

### Если установки не показываются
```bash
npm run seed
# Потом обновить браузер (F5)
```

### Если PDF не генерируется
1. Проверьте консоль браузера (F12)
2. Проверьте логи сервера (`npm run dev`)
3. Убедитесь что у отчета есть siteId и userId

### Если при редактивании не видна "Установка"
1. Если это первое редактирование → создайте новый отчет с установкой
2. Точно ли нажимаете кнопку редактирования (карандаш)?
3. Проверьте консоль браузера (F12) → нет ли красных ошибок?

### Если ошибки в консоли браузера (F12)
- Red errors → ПРОБЛЕМА (нужна отладка)
- Yellow warnings → не критично
- Green logs → все ОК

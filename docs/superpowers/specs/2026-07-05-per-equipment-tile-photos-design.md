# Индивидуальные фото установок в редакторе плитки

## Цель

Сохранить единый редактируемый шаблон плитки, но показывать в фотоблоке собственное изображение каждой установки. Пользователь назначает фотографии локально в редакторе, выбирая установку из списка.

## Пользовательский сценарий

1. Пользователь открывает редактор общего шаблона плитки.
2. В верхней панели выбирает установку для предпросмотра.
3. Добавляет фотоблок или выбирает существующий фотоблок.
4. Загружает JPG, PNG или WebP с компьютера.
5. Файл назначается только выбранной установке и выбранному фотоблоку.
6. При переключении установки холст показывает её собственное фото либо заглушку «Фото не загружено».
7. Размер, положение, границы, режим `contain/cover` и остальные свойства фотоблока остаются общими для всех плиток.

## Архитектура данных

Шаблон продолжает храниться в `localStorage` и описывает только структуру фотоблока. Бинарные файлы хранятся в IndexedDB.

Идентификатор изображения формируется детерминированно из идентификатора установки и идентификатора фотоблока:

```text
equipment-tile-image:{equipmentId}:{blockId}
```

Таким образом один фотоблок общего шаблона получает независимый файл для каждой установки. Ссылка на один общий `assetId` из шаблона больше не используется.

## Интерфейс редактора

- В заголовке редактора появляется поле «Установка для фото» со всеми доступными установками.
- Выбранная установка используется одновременно для предпросмотра плитки и операций загрузки/замены фотографии.
- Кнопка «Добавить фото» создаёт общий фотоблок и загружает файл для выбранной установки.
- В инспекторе отображается имя выбранной установки и кнопка «Заменить фото».
- Если для установки нет файла, фотоблок показывает нейтральную заглушку «Фото не загружено».
- Удаление фотоблока из шаблона удаляет все связанные с ним локальные фотографии после сохранения шаблона.
- Отмена редактирования удаляет только новые файлы, появившиеся в отменённом черновике; сохранённые фотографии не затрагиваются.

## Совместимость

Ранее сохранённое общее фото не назначается автоматически, чтобы исключить случайное копирование изображения одной установки во все остальные. Пользователь один раз назначает его нужной установке заново. Старые записи IndexedDB могут быть очищены при сохранении или сбросе шаблона.

## Ошибки и ограничения

- Разрешены MIME-типы `image/jpeg`, `image/png`, `image/webp`.
- Максимальный размер файла — 12 МБ.
- Ошибка чтения или сохранения показывается рядом с элементом загрузки и не меняет текущую фотографию.
- Если IndexedDB недоступна, используется существующее временное хранилище в памяти; после перезагрузки такие файлы исчезнут.
- На этом этапе фотографии локальны для конкретного браузера. При переносе в приложение интерфейс хранилища будет заменён серверным API без изменения структуры редактора.

## Проверка

- Unit-тест идентификатора изображения подтверждает разделение по установкам и фотоблокам.
- Тест контроллера подтверждает, что загрузка для одной установки не меняет файл другой.
- Тест рендера подтверждает разные изображения в двух плитках одного шаблона.
- Тест редактора проверяет выбор установки, загрузку, замену и заглушку.
- Браузерный сценарий назначает разные фотографии двум установкам, сохраняет шаблон и повторно проверяет их после перезагрузки.
- Отдельно проверяются мобильная панель, отсутствие горизонтального переполнения и доступные подписи элементов.

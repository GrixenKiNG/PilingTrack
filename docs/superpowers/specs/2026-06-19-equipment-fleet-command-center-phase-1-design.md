# Модуль «Установки» как командный центр — дизайн (по образцу preview.html)

Дата: 2026-06-19 (обновл. 2026-06-20)
Статус: согласовано направление, ожидает ревью спеки

## Цель

Реализовать модуль «Установки» **по структуре прототипа `preview.html`**, но на реальных
данных PilingTrack:

- верхняя панель действий, KPI, фильтры, переключатель Плитки/Таблица;
- сетка карточек / таблица парка;
- правая детальная панель с вкладками выбранной установки.

**Светлая тема** (тёмная и тумблер темы не делаем). Модуль встраивается в существующий
admin/ops-shell — боковую навигацию из прототипа не дублируем.

### Два жёстких правила содержимого

1. **«Экономику» удаляем полностью** — вкладки и любых денежных метрик нет.
2. **Честность данных:** где реального источника нет — ставим **заглушку «нет данных»**
   (не выдуманное число). Заглушки оживут на Этапах телеметрии/последующих модулей.

## Единый источник данных

Список и KPI — из **одного места**, того же, что экран `/monitoring` и дашборд:
`getFleetSnapshot()` → `GET /api/monitoring/fleet`
(`src/modules/monitoring/application/queries/fleet-monitoring.service.ts`).

Снапшот (на отчётах) уже даёт: статус `active/expected/idle`, `todayTotals`
{сваи, бурение-м, простой-часы}, объект/оператор/бригада, последний отчёт, и totals парка.

Расширяем сервис инвентарными полями (без второго запроса): `kind`, `inventoryNumber`,
`serialNumber`, `engineHoursTotal`, `nextMaintenanceDate`, `nextMaintenanceAtHours`,
а также `totals.operatorsOnShiftToday`.

Детальные вкладки берут данные из уже существующих источников карточки установки
(`MaintenanceRecord`, `EquipmentDocument`, фото, отчёты) — переиспользуем компоненты
`src/components/piling/admin-equipment/detail/*`, а не пишем заново.

## Вне scope (зафиксировано)

- Карта-GPS, телеметрия (топливо/давление/ошибки/онлайн), износ компонентов → Этап телеметрии (при подключении бокса).
- Экономика — удалена (не возвращаем без отдельного продуктового решения).
- Поле «механик» — нет в схеме.

## Критерии готовности

1. `getMaintenanceFlag` + расширение `fleet-monitoring.service` покрыты тестами; зелёные.
2. Экран по структуре `preview.html` (светлый), KPI/плитки/таблица/правая панель работают.
3. KPI и метрики совпадают с `/monitoring` (единый источник).
4. Все зоны без источника показывают «нет данных»/«ждёт датчик»; нет выдуманных чисел; вкладки «Экономика» нет.
5. Быстрые действия и «Полная карточка» ведут на `/admin/equipment/{id}`.
6. `npm run verify` зелёный; импакт по `getFleetSnapshot`/`FleetCard` без поломки `/monitoring` и дашборда.

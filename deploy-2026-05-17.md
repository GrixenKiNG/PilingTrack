### 2. Освободить диск — ⚠️ КРИТИЧНО

**Билд `app` + `workers` требует ≥ 4 GB свободного места.** На VPS всего 30 GB, и пара образов 2 GB + слой кэша 2 GB легко съедают всё. **Если меньше — билд упадёт `no space left on device` посреди процесса и контейнеры просто исчезнут → прод ляжет.**

```bash
df -h /
# если Avail < 8 GB — обязательно prune:
docker system prune -af --volumes   # самый агрессивный, освобождает 2-3 GB
df -h /
```

**Должно стать ≥ 8 GB свободно перед `docker compose build`.**

Если упало на билде:
```bash
docker compose ps -a                 # увидишь что app/workers нет
docker system prune -af --volumes    # освободить
docker compose build app workers     # пересобрать
docker compose up -d app workers
```

### 4. Пересобрать сервисы — **по одному, с очисткой между ними**

VPS впритык по диску. Билдить `app` и `workers` одновременно (`docker compose build app workers`) рискованно — кэш билдера копится и место кончается посреди. **Делай по очереди и чисти кэш каждый раз.**

```bash
# === APP ===
docker compose stop app
docker compose rm -f app
docker rmi pilingtrack-app:latest 2>/dev/null || true
docker builder prune -af              # ← обязательно перед билдом
df -h /                                # должно быть ≥ 6 GB
docker compose build app
docker compose up -d app
sleep 10
docker compose ps app                  # healthy?

# === WORKERS (если изменился src/server/workers или src/services) ===
docker compose stop workers
docker compose rm -f workers
docker rmi pilingtrack-workers:latest 2>/dev/null || true
docker builder prune -af              # ← снова чистим, app-слои больше не нужны
df -h /
docker compose build workers
docker compose up -d workers

# === WS (если src/server/websocket) ===
# по той же схеме
```

После каждого `up -d` проверяй `docker compose ps <service>` — должен быть `healthy` за 10-30 секунд.

Миграции применит migrate-сервис на старте, но можно явно:

## Риски

| Риск | Вероятность | Митигация |
|---|---|---|
| Билд упадёт по диску | средняя | шаг 2 (prune) |
| Миграция упадёт | низкая | бэкап (шаг 1), additive-only |
| Equipment-passport не подхватит данные | низкая | backfill UPDATE в миграции 140000 |
| Старые операторы залогинены, увидят 401 | средняя | им просто залогиниться заново — JWT TTL 12h, может протухнуть на рестарт |
| Прод-кука сессии не совпадает | низкая | SESSION_COOKIE_NAME не менялся |

    Потоки данных

    Авторизация:

     1 POST /api/auth/login → rateLimiter.check() → bcrypt verify
     2   → jose JWT sign → HttpOnly cookie → Zustand login(user)
     3   ← При перезагрузке: GET /api/auth/me → restore session

    Создание отчёта:

     1 Online:  POST /api/reports/upsert → validate → PrismaRepository.save()
     2   → domain events → outbox enqueue → WS broadcast → alerts
     3 Offline: Dexie (IndexedDB) → outbox.enqueue() → sync при online

    Offline Sync:

     1 runSyncCycle():
     2   1. Push: POST /api/sync/batch → отправка pending outbox
     3   2. Pull: GET /api/sync/updates → delta с сервера
     4   3. Conflict: last-write-wins с updated_at
     5   4. Apply: Dexie bulkPut серверных записей

    ┌──────────────────┬─────────────────────┬──────────────────────────────────────────────────────────┐
    │ Компонент        │ Текущее             │ Масштабирование                                          │
    ├──────────────────┼─────────────────────┼──────────────────────────────────────────────────────────┤
    │ Next.js          │ Standalone output   │ Docker + K8s (Helm chart + ArgoCD готовы)                │
    │ Rate Limit       │ Redis (distributed) │ ✅ Горизонтально масштабируется                          │

    │ PostgreSQL       │ Одна инстанция      │ PgBouncer (connection pooling), партицирование telemetry │
    │ CQRS Projections │ Projection worker   │ Можно распараллелить по типу проекций                    │
    │ Outbox           │ Outbox worker       │ Можно запустить несколько воркеров (partition by ID)     │
    │ PDF генерация    │ Синхронная в API    │ ⚠️ Нужно вынести в фоновую задачу                        │
    │ Кэширование      │ Redis доступен      │ ⚠️ Используется только для rate limit + pub/sub          │

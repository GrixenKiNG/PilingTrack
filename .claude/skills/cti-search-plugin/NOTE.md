# Установка CTI Search Plugin — отдельный шаг

`install.sh` в этой папке — **глобальный** инсталлятор (не проектный): он пишет в
`~/.claude/plugins/`, `~/.claude/commands/` и `~/.claude/claude_desktop_config.json`
пользователя, регистрирует MCP-сервер на уровне **всей машины** (повлияет на все
проекты, не только PilingTrack), требует собственный `BRAVE_SEARCH_API_KEY` и
перезапуск Claude Code.

Файлы скопированы в проект для ревью/доступности, но `install.sh` **не запускался
автоматически** — это решение другого масштаба, чем установка скилов в `.claude/`
проекта. Чтобы реально включить, нужно запустить `bash install.sh` вручную и
осознанно.

# Mode 5 — Self-Healing Documentation CI

Generates everything needed to keep documentation automatically in sync with code:
- Two Cursor `.mdc` rules (Cursor agents apply these on every edit)
- GitHub Actions workflow (drift detection + targeted update on PR and schedule)
- Five Python scripts (analyze → detect → report → update → verify)
- `requirements.txt`
- `docs/DOC_AUTOMATION.md` (living guide for the system)

**Sequence within this mode:**
1. Generate `.cursor/rules/50-hierarchical-documentation.mdc`
2. Generate `.cursor/rules/51-auto-documentation-updates.mdc`
3. Generate `.github/scripts/` Python scripts (all five)
4. Generate `.github/scripts/requirements.txt`
5. Generate `.github/workflows/docs-self-heal.yml`
6. Generate `docs/DOC_AUTOMATION.md`

**Dependencies:** This mode assumes CLAUDE.md and DOC_INDEX.md already exist (generated by
modes 1–4). If they don't, run Mode 0 Express first, then run Mode 5.

## File 1: `.cursor/rules/50-hierarchical-documentation.mdc`

Cursor rule that enforces the three-layer hierarchy on every AI-assisted edit.
`alwaysApply: true` — this fires on every Cursor agent session, not just doc tasks.

```markdown
---
description: Hierarchical documentation system — DOC_INDEX.md → CLAUDE.md → /docs/**
alwaysApply: true
---

# Hierarchical Documentation System

**Portability**: REUSABLE — copy to any project using this documentation stack

## Three-Layer Architecture

```
/DOC_INDEX.md          ← compressed routing layer (read first, ~1–2k tokens)
/CLAUDE.md             ← engineering navigation map (150–300 lines)
/docs/**               ← authoritative domain sub-documents (all detail here)
```

## Rule: Always Start with DOC_INDEX.md

When beginning any task on this repository:

1. Read `/DOC_INDEX.md` — identify which documents are needed
2. Read `/CLAUDE.md` — get architectural context if required
3. Load only the sub-documents listed for your task
4. Do not load sub-documents not listed for your task

## Task-Based Loading

### Understanding the system
```
DOC_INDEX.md → CLAUDE.md → docs/architecture/SYSTEM_OVERVIEW.md
```

### Tracing a runtime flow
```
CLAUDE.md (Section: Core Runtime Flows)
→ docs/architecture/RUNTIME_FLOWS.md
→ docs/architecture/DEPENDENCY_MAP.md  ← only if external services involved
```

### Working on AI prompts or models
```
CLAUDE.md (Section: High-Risk Areas)
→ docs/ai/PROMPTS.md
→ docs/ai/LLM_ARCHITECTURE.md
→ docs/ai/MODEL_GUARDRAILS.md
→ docs/ai/PROMPT_TESTS.md  ← before shipping any change
```

### Changing a data contract or schema
```
CLAUDE.md (Section: High-Risk Areas)
→ docs/architecture/DATA_CONTRACTS.md
→ docs/architecture/DEPENDENCY_MAP.md
```

### Debugging a production failure
```
docs/ai/AI_RUNBOOK.md  ← if AI-related
docs/general/ARCHITECTURE_OVERVIEW.md  ← for infra context
```

### Modifying authentication or authorization
```
CLAUDE.md (Section: High-Risk Areas)
→ docs/general/API_REFERENCE.md#auth
→ docs/ai/MODEL_GUARDRAILS.md#data-privacy  ← if AI involved
```

### Local development or contribution
```
docs/general/README.md → docs/general/CONTRIBUTING.md
```

## Documentation Maintenance Rules

### Rule 1: One Canonical Home Per Topic

Each topic has exactly one authoritative document. All others link to it.

**DO:**
```markdown
<!-- In CLAUDE.md -->
For architecture detail, see docs/architecture/SYSTEM_OVERVIEW.md

<!-- In docs/architecture/SYSTEM_OVERVIEW.md -->
# System Overview
[Full content here]
```

**DON'T:**
```markdown
<!-- In CLAUDE.md -->
# System Overview
[Full content duplicated here — WRONG]
```

### Rule 2: Required Header on Every Sub-Document

```markdown
<!-- Parent: /CLAUDE.md -->
<!-- Index:  /DOC_INDEX.md -->
<!-- Related: /docs/path/related.md, /docs/path/other.md -->
<!-- Read when: [specific scenario] -->

# Document Title

**Scope:** [what this doc covers | what it does NOT cover]
```

### Rule 3: Required Footer on Every Sub-Document

```markdown
---
## Known Gaps / Uncertainties

- [inferred but unverified behavior]
- [area missing from this doc]
- [pointer to related doc for adjacent topics]
```

### Rule 4: Length Gates

| File | Target | Hard Limit |
|------|--------|-----------|
| DOC_INDEX.md | 80–150 lines | 200 lines |
| CLAUDE.md | 150–250 lines | 300 lines |
| Sub-docs | As needed | — |

If CLAUDE.md exceeds 300 lines: content is leaking from sub-docs. Move it back.
If DOC_INDEX.md exceeds 200 lines: content is leaking from CLAUDE.md. Move it back.

### Rule 5: Update Sequence When Adding New Docs

1. Write the sub-document with required header/footer
2. Update CLAUDE.md Documentation Map (Section 7) and Loading Guide (Section 8)
3. Update DOC_INDEX.md Domain table and Task Routing
4. Verify all links resolve

### Rule 6: Never Duplicate Across Layers

If you find yourself writing the same explanation in both CLAUDE.md and a sub-doc:
- Full explanation → sub-doc
- 2-line summary + link → CLAUDE.md

## Validation Checklist (run before committing doc changes)

- [ ] All sub-docs have required header block
- [ ] All sub-docs have Known Gaps footer
- [ ] CLAUDE.md ≤ 300 lines
- [ ] DOC_INDEX.md ≤ 200 lines
- [ ] Every link in CLAUDE.md resolves to an existing file
- [ ] Every link in DOC_INDEX.md resolves to an existing file
- [ ] No topic is documented in detail in more than one file
- [ ] New docs are listed in both CLAUDE.md Section 7 and DOC_INDEX.md Domains

## Changelog

- [PROJECT_DATE]: Rule generated by project-documenter skill
```

## File 2: `.cursor/rules/51-auto-documentation-updates.mdc`

Cursor rule that tells the agent which docs to update when code changes.
`alwaysApply: true` — fires on every Cursor session so the agent never forgets.

```markdown
---
description: Automatically update documentation when code changes affect documented behavior
alwaysApply: true
---

# Automatic Documentation Updates

**Portability**: REUSABLE — copy to any project  
**Override**: HIGH — documentation must stay in sync with code

## When Documentation MUST Be Updated

### API changes
- New endpoint added or removed
- Request/response format changed
- Auth requirement changed
- Error codes changed

→ Update: `docs/general/API_REFERENCE.md`, `docs/architecture/DATA_CONTRACTS.md`
→ Update: `CLAUDE.md` (Section: Entry Points) if new top-level capability

### Architecture changes
- New service or module added
- Component responsibilities changed
- Data flow changed
- Integration point added/removed

→ Update: `docs/architecture/SYSTEM_OVERVIEW.md`
→ Update: `docs/architecture/REPOSITORY_MAP.md`
→ Update: `docs/architecture/RUNTIME_FLOWS.md` (if flows affected)
→ Update: `CLAUDE.md` (Section: Architecture summary) if high-level change

### Configuration changes
- New environment variable
- Config file format changed
- Default values changed

→ Update: `docs/general/README.md` (prerequisites)
→ Update: `docs/general/ARCHITECTURE_OVERVIEW.md` (infra)

### AI / LLM changes
- Prompt added or modified
- Model changed
- Agent workflow changed
- Guardrail added or modified

→ Update: `docs/ai/PROMPTS.md` (version number, template, changes from previous)
→ Update: `docs/ai/PROMPT_TESTS.md` (add/update regression test)
→ Update: `docs/ai/MODEL_GUARDRAILS.md` (if safety impact)
→ Update: `docs/ai/LLM_ARCHITECTURE.md` (if model or pipeline changed)

### Security changes
- Auth mechanism changed
- Trust boundary changed
- Secret handling changed

→ Update: `docs/general/API_REFERENCE.md#auth`
→ Update: `docs/ai/MODEL_GUARDRAILS.md#data-privacy`
→ Update: `CLAUDE.md` (Section: High-Risk Areas)

### Data contract changes
- Schema field added, removed, or renamed
- Event format changed
- Breaking change introduced

→ Update: `docs/architecture/DATA_CONTRACTS.md`
→ Update: `docs/architecture/DEPENDENCY_MAP.md`
→ Update: `CLAUDE.md` (Section: High-Risk Areas)

## Update Workflow

### Step 1: Identify affected documents

Before committing code, run this mental check:

```
Does this change affect:
  □ An API endpoint?              → docs/general/API_REFERENCE.md
  □ System architecture?          → docs/architecture/SYSTEM_OVERVIEW.md
  □ A runtime flow?               → docs/architecture/RUNTIME_FLOWS.md
  □ A data schema or contract?    → docs/architecture/DATA_CONTRACTS.md
  □ An AI prompt or model?        → docs/ai/PROMPTS.md + PROMPT_TESTS.md
  □ Auth or security?             → API_REFERENCE.md#auth + MODEL_GUARDRAILS.md
  □ An external dependency?       → docs/architecture/DEPENDENCY_MAP.md
  □ Local dev or deployment?      → docs/general/README.md
```

### Step 2: Update canonical document first

One topic = one canonical document. Update there first.
Then update any documents that reference it.
Then update CLAUDE.md summary (2 lines max + link) if significant.
Then update DOC_INDEX.md routing table if the task type changes.

### Step 3: Verify the hierarchy remains clean

After updates:
- CLAUDE.md must not exceed 300 lines
- DOC_INDEX.md must not exceed 200 lines
- No detail must exist in CLAUDE.md that should be in a sub-doc

## When NOT to Update CLAUDE.md

**DO update CLAUDE.md for:**
- New high-level capability
- New high-risk area
- New document added to system
- Architecture style change

**DO NOT update CLAUDE.md for:**
- Variable or method renames
- Bug fixes that don't change documented behavior
- Internal refactoring
- Implementation detail changes
- Code comment additions

## Commit Convention

Documentation updates must ship in the same commit as the code change that made them necessary.

```bash
# DO: single commit, code + docs together
git add src/new_feature.py docs/ai/PROMPTS.md docs/architecture/DATA_CONTRACTS.md
git commit -m "feat: add vulnerability enrichment prompt + docs"

# DON'T: code now, docs later
git add src/new_feature.py
git commit -m "feat: add vulnerability enrichment prompt"
# (docs never get updated — this is the failure pattern)
```

## Pre-Commit Validation

```bash
# Check all sub-docs have required parent header
find docs/ -name "*.md" | while read f; do
  grep -q "Parent: /CLAUDE.md" "$f" || echo "MISSING HEADER: $f"
done

# Check CLAUDE.md line count
wc -l CLAUDE.md | awk '{if($1>300) print "WARNING: CLAUDE.md exceeds 300 lines (" $1 ")"}'

# Check DOC_INDEX.md line count
wc -l DOC_INDEX.md | awk '{if($1>200) print "WARNING: DOC_INDEX.md exceeds 200 lines (" $1 ")"}'
```

## Changelog

- [PROJECT_DATE]: Rule generated by project-documenter skill
```

## File 3: `.github/scripts/analyze_codebase_changes.py`

Detects structural code changes (new modules, services, routes, AI components, schema changes).
Writes `.github/codebase-changes.json`.

```python
#!/usr/bin/env python3
"""
Detect structural code changes that may require documentation updates.

Scans git diff for: new modules, services, routes, AI components, config changes, schema changes.
Output: .github/codebase-changes.json
"""

import json
import os
import re
import subprocess
from pathlib import Path
from typing import Dict, List


def get_changed_files() -> List[str]:
    """Get changed files from git — PR diff or last 24h for scheduled runs."""
    try:
        if os.getenv('GITHUB_EVENT_NAME') == 'pull_request':
            result = subprocess.run(
                ['git', 'diff', '--name-only', 'origin/main...HEAD'],
                capture_output=True, text=True, check=True
            )
        else:
            result = subprocess.run(
                ['git', 'log', '--name-only', '--since=24 hours ago', '--pretty=format:'],
                capture_output=True, text=True, check=True
            )
        return list(set(f for f in result.stdout.split('\n') if f.strip()))
    except subprocess.CalledProcessError:
        return []


def scan_file_patterns(files: List[str], patterns: Dict[str, str]) -> Dict[str, List[str]]:
    """Scan file contents for patterns indicating structural changes."""
    results = {key: [] for key in patterns}
    for path_str in files:
        p = Path(path_str)
        if not p.exists() or not p.is_file():
            continue
        try:
            content = p.read_text(errors='ignore')
        except Exception:
            continue
        for key, pattern in patterns.items():
            if re.search(pattern, content):
                results[key].append(path_str)
    return results


def analyze_changes(files: List[str]) -> Dict:
    """Classify changes by type — language-agnostic structural detection."""
    # Detect by file extension groups
    py_files   = [f for f in files if f.endswith('.py')]
    ts_files   = [f for f in files if f.endswith(('.ts', '.tsx', '.js', '.jsx'))]
    kt_files   = [f for f in files if f.endswith('.kt')]
    go_files   = [f for f in files if f.endswith('.go')]
    rb_files   = [f for f in files if f.endswith('.rb')]
    other_src  = [f for f in files if any(f.endswith(e) for e in ('.cs', '.java', '.rs', '.cpp', '.c'))]

    config_files = [f for f in files if any(x in f for x in [
        'application.yml', 'application.properties', '.env', 'config.', 'settings.', 'pyproject.toml',
        'package.json', 'Dockerfile', 'docker-compose', 'terraform', '.github/workflows'
    ])]
    schema_files = [f for f in files if any(x in f.lower() for x in ['schema', 'migration', 'model', 'proto'])]
    ai_files = [f for f in files if any(x in f.lower() for x in ['prompt', 'llm', 'ai/', 'agent', 'guardrail'])]
    doc_files = [f for f in files if f.endswith('.md') and ('docs/' in f or 'CLAUDE' in f or 'DOC_INDEX' in f)]

    # Pattern-based detection across all source files
    all_src = py_files + ts_files + kt_files + go_files + rb_files + other_src
    patterns = {
        'new_routes':      r'(@app\.(get|post|put|delete|patch)|@Router|@RestController|router\.(get|post)|Route\(|@route)',
        'new_services':    r'(@Service|@Injectable|class \w+Service|def \w+_service|@service)',
        'new_models':      r'(class \w+Model|@Entity|data class|@dataclass|type \w+ struct|Schema\()',
        'new_ai_prompts':  r'(prompt|PROMPT|system_prompt|user_prompt|messages\s*=\s*\[)',
        'new_agents':      r'(Agent|@Tool|McpTool|tool_use|function_call)',
        'auth_changes':    r'(@auth|@require_auth|authenticate|authorize|JWT|OAuth|bearer)',
    }
    detected = scan_file_patterns(all_src, patterns)

    has_changes = bool(
        detected['new_routes'] or detected['new_services'] or
        detected['new_models'] or detected['new_ai_prompts'] or
        detected['new_agents'] or detected['auth_changes'] or
        config_files or schema_files
    )

    return {
        'has_changes': has_changes,
        'changed_files': files,
        'by_type': {
            'routes':    detected['new_routes'],
            'services':  detected['new_services'],
            'models':    detected['new_models'],
            'ai_prompts': detected['new_ai_prompts'],
            'agents':    detected['new_agents'],
            'auth':      detected['auth_changes'],
            'config':    config_files,
            'schema':    schema_files,
            'ai_files':  ai_files,
            'docs':      doc_files,
        }
    }


def main():
    print("Analyzing codebase changes...")
    files = get_changed_files()
    print(f"  {len(files)} changed files")

    changes = analyze_changes(files)
    Path('.github').mkdir(exist_ok=True)
    Path('.github/codebase-changes.json').write_text(json.dumps(changes, indent=2))
    print(f"  has_changes: {changes['has_changes']}")

    if os.getenv('GITHUB_OUTPUT'):
        with open(os.getenv('GITHUB_OUTPUT'), 'a') as f:
            f.write(f"has_changes={str(changes['has_changes']).lower()}\n")


if __name__ == '__main__':
    main()
```

## File 4: `.github/scripts/detect_documentation_drift.py`

Uses Claude to compare codebase changes against existing docs and classify drift by severity.
Writes `.github/drift-analysis.json`.

```python
#!/usr/bin/env python3
"""
Use AI to detect documentation drift — compare codebase changes to current docs.

Input:  .github/codebase-changes.json
Output: .github/drift-analysis.json
Env:    ANTHROPIC_API_KEY (required) or OPENAI_API_KEY (fallback)
"""

import json
import os
from pathlib import Path
from typing import Dict

try:
    import anthropic
    HAS_ANTHROPIC = True
except ImportError:
    HAS_ANTHROPIC = False

try:
    import openai
    HAS_OPENAI = True
except ImportError:
    HAS_OPENAI = False

DRIFT_PROMPT = """You are a senior staff engineer maintaining the documentation system for this repository.

The documentation uses a three-layer hierarchy:
  Layer 1: DOC_INDEX.md — compressed routing layer
  Layer 2: CLAUDE.md — engineering navigation map
  Layer 3: /docs/** — authoritative sub-documents

CODEBASE CHANGES DETECTED
--------------------------
{changes_summary}

CURRENT DOCUMENTATION STATE
----------------------------
DOC_INDEX.md (first 1500 chars):
{doc_index_summary}

CLAUDE.md (first 1500 chars):
{claude_summary}

TASK
----
Detect documentation drift. Be conservative — only flag drift where there is clear evidence
that the code change is NOT reflected in the documentation.

SEVERITY CLASSIFICATION
-----------------------
high:   New entry point, removed service, changed runtime flow, new AI component, auth change
medium: New API endpoint, new service, configuration change, schema change
low:    Minor feature addition, internal refactoring with external API unchanged

DO NOT FLAG:
- Cosmetic changes (formatting, comments, variable renames)
- Changes that are already reflected in the docs
- Internal refactoring that does not change documented behavior

OUTPUT: Return ONLY valid JSON matching this exact schema:
{{
  "has_drift": true,
  "severity": "high|medium|low",
  "drift_items": [
    {{
      "category": "architecture|api|ai|security|config|schema",
      "description": "Brief description of what changed and what is outdated",
      "affected_docs": ["docs/path/to/file.md"],
      "evidence": "Specific evidence from the codebase change list",
      "severity": "high|medium|low"
    }}
  ],
  "unverified_behavior": ["behavior inferred but not confirmed"],
  "recommendations": ["Specific doc update needed — file + section"]
}}
"""


def load_changes() -> Dict:
    p = Path('.github/codebase-changes.json')
    return json.loads(p.read_text()) if p.exists() else {'has_changes': False}


def load_doc_summaries() -> Dict[str, str]:
    out = {}
    for path, key in [('DOC_INDEX.md', 'doc_index'), ('CLAUDE.md', 'claude')]:
        p = Path(path)
        out[key] = p.read_text()[:1500] if p.exists() else 'Not found'
    return out


def changes_to_summary(changes: Dict) -> str:
    lines = []
    by_type = changes.get('by_type', {})
    for kind, files in by_type.items():
        if files:
            lines.append(f"  {kind}: {len(files)} file(s) — {', '.join(files[:3])}")
    return '\n'.join(lines) or 'No structural changes detected'


def call_claude(prompt: str) -> Dict:
    client = anthropic.Anthropic(api_key=os.environ['ANTHROPIC_API_KEY'])
    msg = client.messages.create(
        model='claude-sonnet-4-6',
        max_tokens=4096,
        messages=[{'role': 'user', 'content': prompt}]
    )
    text = msg.content[0].text
    start, end = text.find('{'), text.rfind('}') + 1
    return json.loads(text[start:end]) if start >= 0 else empty_result()


def call_openai(prompt: str) -> Dict:
    client = openai.OpenAI(api_key=os.environ['OPENAI_API_KEY'])
    resp = client.chat.completions.create(
        model='gpt-4-turbo-preview',
        messages=[
            {'role': 'system', 'content': 'Senior staff engineer. Return only JSON.'},
            {'role': 'user', 'content': prompt}
        ],
        response_format={'type': 'json_object'}
    )
    return json.loads(resp.choices[0].message.content)


def empty_result() -> Dict:
    return {'has_drift': False, 'severity': 'low', 'drift_items': [],
            'unverified_behavior': [], 'recommendations': []}


def main():
    print("Detecting documentation drift...")
    changes = load_changes()

    if not changes.get('has_changes'):
        print("  No structural changes — skipping drift detection")
        Path('.github').mkdir(exist_ok=True)
        Path('.github/drift-analysis.json').write_text(json.dumps(empty_result(), indent=2))
        if os.getenv('GITHUB_OUTPUT'):
            with open(os.getenv('GITHUB_OUTPUT'), 'a') as f:
                f.write("has_drift=false\nseverity=low\n")
        return

    summaries = load_doc_summaries()
    prompt = DRIFT_PROMPT.format(
        changes_summary=changes_to_summary(changes),
        doc_index_summary=summaries['doc_index'],
        claude_summary=summaries['claude']
    )

    result = empty_result()
    try:
        if HAS_ANTHROPIC and os.getenv('ANTHROPIC_API_KEY'):
            print("  Using Claude for analysis...")
            result = call_claude(prompt)
        elif HAS_OPENAI and os.getenv('OPENAI_API_KEY'):
            print("  Using GPT-4 for analysis...")
            result = call_openai(prompt)
        else:
            print("  WARNING: No AI API available — using heuristic fallback")
            result = {
                'has_drift': changes.get('has_changes', False),
                'severity': 'medium',
                'drift_items': [{
                    'category': 'unknown',
                    'description': 'Structural changes detected; AI analysis unavailable',
                    'affected_docs': ['DOC_INDEX.md', 'CLAUDE.md'],
                    'evidence': 'See .github/codebase-changes.json',
                    'severity': 'medium'
                }],
                'unverified_behavior': [],
                'recommendations': ['Manual documentation review required']
            }
    except Exception as e:
        print(f"  ERROR: {e}")

    Path('.github').mkdir(exist_ok=True)
    Path('.github/drift-analysis.json').write_text(json.dumps(result, indent=2))
    print(f"  has_drift: {result['has_drift']}, severity: {result['severity']}")

    if os.getenv('GITHUB_OUTPUT'):
        with open(os.getenv('GITHUB_OUTPUT'), 'a') as f:
            f.write(f"has_drift={str(result['has_drift']).lower()}\n")
            f.write(f"severity={result['severity']}\n")


if __name__ == '__main__':
    main()
```

## File 5: `.github/scripts/generate_drift_report.py`

Converts `drift-analysis.json` into a human-readable markdown report for PR comments.
Writes `.github/drift-report.md`.

```python
#!/usr/bin/env python3
"""
Generate human-readable markdown drift report for PR comments and artifacts.

Input:  .github/drift-analysis.json
Output: .github/drift-report.md
"""

import json
from pathlib import Path


SEVERITY_EMOJI = {'high': '🔴', 'medium': '🟡', 'low': '🟢'}
CATEGORY_LABELS = {
    'architecture': '🏗️ Architecture',
    'api':          '🔌 API',
    'ai':           '🤖 AI / LLM',
    'security':     '🔒 Security',
    'config':       '⚙️ Configuration',
    'schema':       '📐 Data Contracts',
    'unknown':      '❓ Unknown',
}


def load_analysis() -> dict:
    p = Path('.github/drift-analysis.json')
    return json.loads(p.read_text()) if p.exists() else {}


def generate_report(analysis: dict) -> str:
    if not analysis.get('has_drift'):
        return "✅ **Documentation is up to date.** No drift detected."

    severity = analysis.get('severity', 'low')
    emoji = SEVERITY_EMOJI.get(severity, '🟡')
    lines = [
        f"## {emoji} Documentation Drift Detected — {severity.upper()} severity",
        "",
        "Documentation must be updated before merging.",
        "",
    ]

    # Group drift items by category
    items_by_cat: dict = {}
    for item in analysis.get('drift_items', []):
        cat = item.get('category', 'unknown')
        items_by_cat.setdefault(cat, []).append(item)

    for cat, items in items_by_cat.items():
        lines.append(f"### {CATEGORY_LABELS.get(cat, cat.title())}")
        lines.append("")
        for item in items:
            sev_e = SEVERITY_EMOJI.get(item.get('severity', 'low'), '🟡')
            lines.append(f"{sev_e} **{item.get('description', 'Unknown drift')}**")
            if item.get('affected_docs'):
                lines.append(f"   - **Affected docs:** {', '.join(f'`{d}`' for d in item['affected_docs'])}")
            if item.get('evidence'):
                ev = item['evidence'][:120]
                lines.append(f"   - **Evidence:** {ev}{'...' if len(item['evidence']) > 120 else ''}")
            lines.append("")

    if analysis.get('unverified_behavior'):
        lines += ["### ⚠️ Unverified Behavior", ""]
        for b in analysis['unverified_behavior']:
            lines.append(f"- {b}")
        lines.append("")

    if analysis.get('recommendations'):
        lines += ["### 💡 Recommended Updates", ""]
        for r in analysis['recommendations']:
            lines.append(f"- {r}")
        lines.append("")

    lines += [
        "---",
        "",
        "**How to fix:** Update the affected documents listed above, then re-run the CI check.",

## File 6: `.github/scripts/update_documentation.py`

Uses Claude to surgically update only the affected sections of impacted docs.
Never regenerates entire documents. Preserves accurate content.
Writes updated files in place.

```python
#!/usr/bin/env python3
"""
Surgically update only affected documentation sections.

Rules:
- NEVER regenerate an entire document
- Only update sections directly affected by the drift item
- Preserve all existing accurate content
- Maintain document structure, cross-links, and header/footer blocks

Input:  .github/drift-analysis.json + existing doc files
Output: Updated doc files (in place)
Env:    ANTHROPIC_API_KEY (required)
"""

import json
import os
from pathlib import Path
from typing import Dict, List, Optional

try:
    import anthropic
    HAS_ANTHROPIC = True
except ImportError:
    HAS_ANTHROPIC = False

UPDATE_PROMPT = """You are a senior staff engineer performing a conservative, targeted documentation update.

DRIFT ITEM
----------
Category:      {category}
Description:   {description}
Affected docs: {affected_docs}
Evidence:      {evidence}
Severity:      {severity}

CURRENT DOCUMENT: {doc_path}
-----------------------
{current_content}

TASK
----
Update ONLY the sections of this document that are directly outdated by the drift item above.

STRICT RULES:
1. Preserve ALL content that is still accurate
2. Update ONLY the sections affected by this specific drift item
3. Maintain the document's existing structure, headers, footers
4. Maintain all cross-links (<!-- Parent: ... -->, <!-- Related: ... -->)
5. Maintain the Known Gaps / Uncertainties footer
6. Do NOT add speculative content — only document what is confirmed by the evidence
7. Mark any behavior inferred (not confirmed) as: *Unverified — confirm from codebase*
8. Do NOT regenerate the entire document
9. Keep the document within its established scope

If this document does NOT need updating for this drift item, respond with exactly: NO_UPDATE_NEEDED

Otherwise, respond with the complete updated document content (not a diff — the full file).
"""


def load_drift() -> Dict:
    p = Path('.github/drift-analysis.json')
    return json.loads(p.read_text()) if p.exists() else {}


def update_doc(doc_path: str, drift_item: Dict, client: 'anthropic.Anthropic') -> Optional[str]:
    """Return updated content string, or None if no update needed / failed."""
    p = Path(doc_path)
    if not p.exists():
        print(f"  SKIP: {doc_path} not found")
        return None

    content = p.read_text()
    prompt = UPDATE_PROMPT.format(
        category=drift_item.get('category', 'unknown'),
        description=drift_item.get('description', ''),
        affected_docs=', '.join(drift_item.get('affected_docs', [])),
        evidence=drift_item.get('evidence', ''),
        severity=drift_item.get('severity', 'low'),
        doc_path=doc_path,
        current_content=content[:10000],
    )

    try:
        msg = client.messages.create(
            model='claude-sonnet-4-6',
            max_tokens=8192,
            messages=[{'role': 'user', 'content': prompt}]
        )
        response = msg.content[0].text.strip()
        if response == 'NO_UPDATE_NEEDED':
            return None
        return response
    except Exception as e:
        print(f"  ERROR updating {doc_path}: {e}")
        return None


def main():
    print("Updating documentation (targeted, conservative)...")

    if not HAS_ANTHROPIC or not os.getenv('ANTHROPIC_API_KEY'):
        print("  ERROR: ANTHROPIC_API_KEY not set — cannot update")
        return

    drift = load_drift()
    if not drift.get('has_drift'):
        print("  No drift — nothing to update")
        return

    client = anthropic.Anthropic(api_key=os.environ['ANTHROPIC_API_KEY'])
    updated: List[str] = []
    skipped: List[str] = []

    for item in drift.get('drift_items', []):
        for doc in item.get('affected_docs', []):
            print(f"  Processing {doc} for: {item.get('description', '')[:60]}...")
            new_content = update_doc(doc, item, client)
            if new_content:
                Path(doc).write_text(new_content)
                updated.append(doc)
                print(f"  ✓ Updated {doc}")
            else:
                skipped.append(doc)
                print(f"  – No change needed for {doc}")

    print(f"\nSummary: {len(updated)} updated, {len(skipped)} unchanged")

    # Write update summary for CI
    summary = {'updated': updated, 'skipped': skipped}
    Path('.github').mkdir(exist_ok=True)
    Path('.github/update-summary.json').write_text(json.dumps(summary, indent=2))


if __name__ == '__main__':
    main()
```

## File 7: `.github/scripts/verify_documentation_integrity.py`

Verifies cross-links, length gates, required headers/footers, and token counts after updates.
Exits with code 1 if critical issues found (blocks CI).

```python
#!/usr/bin/env python3
"""
Verify documentation integrity after updates.

Checks:
- All cross-links in CLAUDE.md and DOC_INDEX.md resolve
- CLAUDE.md ≤ 300 lines, DOC_INDEX.md ≤ 200 lines
- All sub-docs under /docs have required header and footer
- Token count estimates for top-level files

Exits 1 if critical checks fail (broken links, length gate violations).
"""

import re
import sys
from pathlib import Path
from typing import Dict, List, Tuple


def extract_md_links(text: str) -> List[str]:
    return [m[1] for m in re.findall(r'\[([^\]]+)\]\(([^\)#)]+)', text)]


def check_links(file_path: str) -> List[str]:
    """Return list of broken link paths."""
    p = Path(file_path)
    if not p.exists():
        return []
    broken = []
    for link in extract_md_links(p.read_text()):
        if link.startswith('http') or link.startswith('mailto'):
            continue
        # Resolve relative to repo root
        resolved = Path(link.lstrip('/'))
        if not resolved.exists():
            broken.append(link)
    return broken


def check_length_gates() -> List[Tuple[str, int, int]]:
    """Return (file, actual_lines, limit) for files over their limit."""
    gates = [('CLAUDE.md', 300), ('DOC_INDEX.md', 200)]
    violations = []
    for fname, limit in gates:
        p = Path(fname)
        if p.exists():
            lines = len(p.read_text().splitlines())
            if lines > limit:
                violations.append((fname, lines, limit))
    return violations


def check_sub_doc_headers() -> List[str]:
    """Return list of sub-docs missing the required parent header."""
    missing = []
    for p in Path('docs').rglob('*.md') if Path('docs').exists() else []:
        content = p.read_text()
        if 'Parent: /CLAUDE.md' not in content and '**Parent document:**' not in content:
            missing.append(str(p))
    return missing


def check_sub_doc_footers() -> List[str]:
    """Return list of sub-docs missing the Known Gaps footer."""
    missing = []
    for p in Path('docs').rglob('*.md') if Path('docs').exists() else []:
        if 'Known Gaps' not in p.read_text():
            missing.append(str(p))
    return missing


def estimate_tokens(text: str) -> int:
    return len(text) // 4


def main():
    print("Verifying documentation integrity...\n")
    critical_failures = False

    # 1. Cross-link check
    for fname in ['CLAUDE.md', 'DOC_INDEX.md']:
        broken = check_links(fname)
        if broken:
            print(f"BROKEN LINKS in {fname}:")
            for link in broken:
                print(f"  ✗ {link}")
            critical_failures = True
        else:
            print(f"✓ Links valid: {fname}")

    # 2. Length gate check
    violations = check_length_gates()
    if violations:
        for fname, actual, limit in violations:
            print(f"GATE VIOLATION: {fname} is {actual} lines (limit: {limit})")
            print(f"  → Content is leaking from sub-documents. Move detail to appropriate sub-doc.")
        critical_failures = True
    else:
        print("✓ Length gates: CLAUDE.md and DOC_INDEX.md within limits")

    # 3. Sub-doc header check (warning, not critical)
    missing_headers = check_sub_doc_headers()
    if missing_headers:
        print(f"\nWARNING: {len(missing_headers)} sub-doc(s) missing parent header:")
        for f in missing_headers[:5]:
            print(f"  ! {f}")
    else:
        print("✓ Sub-doc headers: all present")

    # 4. Sub-doc footer check (warning, not critical)
    missing_footers = check_sub_doc_footers()
    if missing_footers:
        print(f"\nWARNING: {len(missing_footers)} sub-doc(s) missing Known Gaps footer:")
        for f in missing_footers[:5]:
            print(f"  ! {f}")
    else:
        print("✓ Sub-doc footers: all present")

    # 5. Token count report
    print("\nToken estimates (top-level files):")
    for fname in ['DOC_INDEX.md', 'CLAUDE.md']:
        p = Path(fname)
        if p.exists():
            tokens = estimate_tokens(p.read_text())
            flag = "✓" if tokens < 3000 else "⚠"
            print(f"  {flag} {fname}: ~{tokens} tokens")

    if critical_failures:
        print("\nINTEGRITY CHECK FAILED — fix issues above before merging")
        sys.exit(1)
    else:
        print("\n✓ Documentation integrity verified")


if __name__ == '__main__':
    main()
```

## What This System Does

Whenever code changes, the self-healing system:

1. **Detects** structural changes (new routes, services, schemas, AI components)
2. **Identifies drift** — documentation that no longer matches the codebase
3. **Reports drift** on PRs as a comment with specific affected files
4. **Updates docs** (nightly or manual) — surgically, not by regenerating everything
5. **Verifies integrity** — broken links, length gate violations, missing headers
6. **Blocks merge** on high-severity drift (requires docs update before merge)

```
Code commit / PR
       ↓
analyze_codebase_changes.py   ← what changed structurally?
       ↓
detect_documentation_drift.py ← is documentation outdated?
       ↓
generate_drift_report.py      ← human-readable report
       ↓
update_documentation.py       ← (update mode only) targeted section updates
       ↓
verify_documentation_integrity.py ← links, length gates, headers
       ↓
PR comment + artifact         ← (check mode)
OR auto-commit                ← (update mode: nightly + manual)
```
